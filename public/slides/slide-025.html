CLI Tool Approval System

When the agent wants to run a command, you get three options:

  1. Yes — Allow this specific command, this time only
  2. Yes, and approve for the session — Allow all uses of this tool
  3. No (Esc) — Cancel and give different instructions

Command-line flags:

--allow-all-tools                  # No prompts at all
--allow-tool=&#x27;shell(git)'          # Allow specific tool
--deny-tool=&#x27;shell(rm)'            # Block specific tool
The CLI has a granular approval system. You can approve individual commands, approve a tool for the whole session, or block it entirely. The command-line flags let you pre-configure permissions — for example, allow git commands but block `rm`. You can even combine: allow all tools except `rm -rf`. This gives you fine-grained control over what the agent can do.
