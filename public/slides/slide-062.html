"Hooks are useful when you need more control than skills or custom instructions can provide."
Hooks are the right choice when you need deterministic enforcement — not guidance, but guarantees. If you just want coding conventions followed, custom instructions are sufficient. If you want a repeatable workflow, use a skill. If you need external capabilities, use MCP. But if you need to guarantee that certain commands are always blocked, or that every action is logged, hooks are the tool for the job.
