Cloud Agent — How It Works

👤 YOU
"Fix issue #42"
☁️ CLOUD AGENT
1. Spins up ephemeral environment
2. Clones repository
3. Researches codebase
4. Creates implementation plan
5. Writes code iteratively
6. Runs tests & linters
7. Runs security scans (CodeQL)
8. Self-reviews with Copilot review
9. Creates copilot/ branch
10. Opens draft PR
Ephemeral GitHub Actions Runner
👥 TEAM
Reviews PR → Approves → Merges
Here's the step-by-step flow. The agent creates a fresh, ephemeral environment — like a clean CI container. It clones your repo, researches the code, creates a plan, and then iteratively writes code, tests, and scans for security issues. It even runs Copilot code review on its own changes before opening the PR. Everything happens on a `copilot/` branch — it can't push to main or merge. A human always reviews and merges.
