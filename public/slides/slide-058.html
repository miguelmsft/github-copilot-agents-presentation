The preToolUse Hook — Most Powerful

The only hook that can directly control agent behavior.

It can return:

  • "deny" → Tool is blocked, agent gets the reason ✅ Works everywhere
  • "allow" → Tool proceeds ⚠️ Cloud/CLI: not currently processed
  • "ask" → User prompted for confirmation ⚠️ Cloud/CLI: not currently processed
{
  "permissionDecision": "deny",
  "permissionDecisionReason": "Privilege escalation not allowed."
}
The preToolUse hook is the star of the show. When the agent wants to run a bash command, edit a file, or call any tool, the preToolUse hook fires first. Your script receives the tool name and arguments as JSON on stdin. You check the arguments against your policies and return a decision. Important caveat: on the cloud agent and CLI, only "deny" actually works right now. If you return "allow" or "ask", the tool proceeds as if no hook ran. So in practice, your hooks should focus on blocking dangerous actions — don't rely on "allow" or "ask" for enforcement on those surfaces. VS Code may handle these values more fully, but check the latest docs.
