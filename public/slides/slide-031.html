Here's a simple decision tree. If your task needs IDE context — like debugging a test failure or using a VS Code extension — use a local agent. If it's well-defined and you want team collaboration through a PR, use the cloud agent. If it's well-defined but you just want it done in the background on your machine, use the CLI. And if you're not sure yet, start local to clarify the requirements, then hand off to CLI or cloud. You can seamlessly transition between agent types.
