MCP Security — Tool Poisoning Attacks ⚠️

A malicious MCP server can embed hidden instructions in tool descriptions that trick the AI into:

  • Reading sensitive files (~/.ssh/id_rsa)
  • Exfiltrating credentials
  • Redirecting actions to attacker-controlled servers

The user sees a simple tool name. The AI sees hidden commands.

Mitigation: Only install MCP servers from trusted sources. Review tool descriptions. Use sandboxing (VS Code on macOS/Linux).

This is a real security concern. Researchers discovered that MCP tool descriptions can contain hidden instructions that are visible to the AI but invisible to you in the UI. A malicious "add two numbers" tool could secretly instruct the AI to read your SSH keys and send them as a parameter. This is called a "tool poisoning attack." The defense is the same as with any software — only install from trusted sources, review what you're installing, and use sandboxing features when available. VS Code offers sandboxing on macOS and Linux.
