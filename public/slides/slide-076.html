Common Pitfalls

  • ❌ Running CLI from home directory — exposes too many files to the agent
  • ❌ Using --allow-all-tools in untrusted repos — the agent can run anything you can
  • ❌ Approving a tool "for the session" — means ALL uses (e.g., approving rm allows rm -rf ./*)
  • ❌ Overloading context — too many tools or instructions reduces quality
  • ❌ Skipping setup steps — the cloud agent wastes time discovering dependencies
  • ❌ Overly broad issues — "Rewrite the codebase" will not go well
A few things to watch out for. Always launch the CLI from your project directory, not your home directory — otherwise the agent can see all your files. Be careful with tool approvals — approving `rm` for the session means the agent can delete anything without asking again. Don't overload with context — too many MCP servers or instructions can actually reduce quality. And for cloud agents, always create a `copilot-setup-steps.yml` workflow to pre-install dependencies.
