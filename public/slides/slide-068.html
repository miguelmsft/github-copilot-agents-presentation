You are a testing specialist. Your responsibilities:

- Analyze existing tests for coverage gaps
- Write unit, integration, and e2e tests
- Ensure tests are isolated and deterministic
- Focus only on test files — never modify production code
Custom agents go a step further — they define a specialized persona with tool restrictions. This test specialist can only read, edit, and search — it can't run shell commands or browse the web. Its instructions limit it to test files only. You select it from the agent dropdown in VS Code or the agents panel on GitHub.com. Custom agents are like team roles — you might have a test specialist, a security reviewer, and a planner, each with different capabilities and constraints.
