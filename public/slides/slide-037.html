Two Transport Modes

stdio (Local)Streamable HTTP (Remote)
In plain termsServer runs as a program on your computerServer runs on the internet
How they talkText piped in/out (like typing into a program)Web requests (like a browser calling an API)
Best forLocal tools on your machineCloud APIs and shared services
SecurityInherits your local permissionsSupports OAuth 2.1 login
SetupSimple — just a commandRequires URL + possibly auth
Two ways MCP servers connect. Stdio — which stands for "standard input/output" — means the server runs as a local program on your machine. The host sends messages in, the server sends responses out — like typing commands into a program. Most MCP servers you'll encounter use this mode because it's simple and secure. The other mode, Streamable HTTP, is for remote servers hosted on the internet. Think of it like calling a web API. Remote servers support OAuth login for authentication. For beginners, you'll mostly work with stdio servers.
