MCP Hosts, Clients, Servers

  • Host = The AI application you interact with (VS Code, Copilot CLI, Claude)
  • Client = A connector created by the host for each server connection
  • Server = A program that provides tools, resources, and prompts

Two ways servers connect:

  • stdio — Server runs as a local subprocess (best for local tools)
  • Streamable HTTP — Server runs remotely over the network (best for cloud APIs)
Important distinction: the host is what you interact with, while clients are internal protocol components — one per server. Servers can run locally on your machine via stdio, or remotely over HTTP. Local servers are simpler to set up and more secure. Remote servers are needed when the service lives in the cloud. Most community MCP servers use stdio.
