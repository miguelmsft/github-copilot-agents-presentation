Hook Configuration Format

.github/hooks/security-hooks.json:

{
  "version": 1,
  "hooks": {
    "preToolUse": [
      {
        "type": "command",
        "bash": "./scripts/security-check.sh",
        "powershell": "./scripts/security-check.ps1",
        "cwd": ".github/hooks",
        "timeoutSec": 15
      }
    ],
    "sessionStart": [
      {
        "type": "command",
        "bash": "echo 'Policy hooks active'",
        "timeoutSec": 5
      }
    ]
  }
}
Hooks are configured as JSON files in `.github/hooks/`. Each hook type gets an array of commands that run in order. You specify separate bash and PowerShell commands for cross-platform support. The `cwd` field sets the working directory, and `timeoutSec` prevents hooks from hanging — default is 30 seconds. You can have multiple JSON files — they're all loaded. For the cloud agent, these files must be on the default branch.
