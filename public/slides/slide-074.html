Copilot Memory — Persistent Knowledge

Copilot can learn and remember things about your repository across sessions.

  • Memories are small, scoped facts Copilot deduces as it works
  • Stored per-repository — not shared across repos
  • Auto-validated against the current codebase before use
  • Auto-expired after 28 days if not refreshed
  • Cross-feature: a memory created in the CLI can be used by code review
"Copilot can develop a persistent understanding of a repository by storing 'memories.'"
One more customization feature worth knowing: Copilot Memory. Unlike custom instructions that you write manually, memories are things Copilot learns on its own as it works — like "this project uses Tailwind CSS" or "tests go in the `__tests__` folder." These persist across sessions within the same repository and expire after 28 days. Copilot validates memories against the current codebase before using them, so they stay accurate. This is currently in public preview. It reduces the need to repeat context in every session. Transition: Now let's talk about best practices.
