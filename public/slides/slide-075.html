Best Practices: The WRAP Framework

GitHub engineers developed WRAP for effective agent usage:

  • W — Write effective issues: Describe tasks as if for a new team member. Include context, examples, and acceptance criteria.
  • R — Refine your instructions: Iterate on custom instructions, skills, and agent definitions over time.
  • A — Atomic tasks: Break large problems into small, independent, well-defined tasks.
  • P — Pair with the agent: Leverage agent strengths (tireless, repetitive work) and human strengths (ambiguity, "why" questions).
The WRAP framework comes from GitHub engineers who use cloud agents every day. Write issues like you're onboarding a new developer — be explicit. Refine your instructions iteratively — if the agent keeps making the same mistake, add it to your instructions. Keep tasks atomic — don't ask the agent to rewrite your entire codebase. And pair effectively — the agent is great at repetitive execution, but you're better at understanding context and making judgment calls.
