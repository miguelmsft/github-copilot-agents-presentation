.github/copilot-instructions.md — applies to every interaction:

# Project Overview
This is a React/Node.js web app using MongoDB.

## Coding Standards
- Use TypeScript with strict mode
- Use functional components with hooks
- Follow PEP 8 for Python scripts
- Write JSDoc comments for public functions

## Architecture
- /src — Frontend React code
- /server — Node.js backend
- /docs — API documentation

Also: path-specific instructions (.github/instructions/*.instructions.md) with glob patterns.

Custom instructions are the simplest customization — a Markdown file that gives Copilot context about your project. It's included in every interaction, so it should contain broadly applicable information: your tech stack, coding standards, architecture overview. Path-specific instructions let you have different rules for different file types — TypeScript conventions for `.ts` files, Python conventions for `.py` files. These use glob patterns in the frontmatter.
