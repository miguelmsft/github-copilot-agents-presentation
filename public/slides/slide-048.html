Progressive Disclosure

Skills use a smart three-tier loading strategy:

1
CATALOG
Name + Description loaded at session start for ALL available skills
~50-100 tokens/skill
2
INSTRUCTIONS
Full SKILL.md body loaded ONLY when the skill is activated (matched to task)
<5000 tokens
3
RESOURCES
Scripts, references, assets loaded ONLY when instructions reference them
varies

20 installed skills ≠ 20× the context cost!

This is the key design insight. If you have 20 skills installed, the agent doesn't pay the token cost of loading all 20 into context. At startup, it only loads the names and descriptions — maybe 50-100 tokens per skill. When it encounters a task that matches a skill's description, it loads the full instructions. And any referenced scripts or files are only loaded when the instructions explicitly refer to them. This keeps the context window efficient.
