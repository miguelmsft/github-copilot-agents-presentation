Tools — The Most Important Primitive

Each tool has a name, a description, and a schema (a blueprint describing what inputs it needs):

{
  "name": "searchFlights",
  "description": "Search for available flights",
  "inputSchema": {
    "type": "object",
    "properties": {
      "origin": { "type": "string" },
      "destination": { "type": "string" },
      "date": { "type": "string", "format": "date" }
    },
    "required": ["origin", "destination", "date"]
  }
}

The agent reads the description, decides when to call it, and fills in the inputs.

Here's what a tool definition looks like. The name identifies it. The description tells the AI when to use it — think of it as a label. The input schema is a blueprint — it says "this tool needs an origin, a destination, and a date, all as text strings." JSON Schema is just a standard way to describe what data looks like — if you've ever filled out a form with required fields, you already understand the concept. When the agent encounters a task that matches this description, it calls the tool with the right arguments.
